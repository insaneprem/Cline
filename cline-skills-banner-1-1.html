Skills

On-Demand Specialist Knowledge for Tizen

Without Skills, you re-explain your Tizen review standards on every session — or keep a bloated rules file that fires whether you're auditing C memory or just fixing a typo. Skills load only when the request matches. Silence the rest.

◎
🎯
One Skill, One Domain
tizen-c-review stays separate from tizen-csharp-nui. Toggle independently. Right context, every time.
◎
🗂️
Route, Don't Cram
SKILL.md routes to docs/ files. Cline reads only what the current file needs — C or C# never bleed into each other.
◎
✍️
Write How You'd Type It
"Review Tizen C for memory leaks" fires every time. "Tizen helper" never does. Your natural phrasing is your trigger.
SKILL.md — tizen-code-review
---
name:       tizen-code-review
description: Review Tizen native C, C++, or C# NUI code
             for memory safety, privilege usage & TIDL validity.
             Use when asked to review, audit, or inspect.
---
# Tizen Code Review
## Language Routing
C native    → docs/c-native-review.md   # malloc, privilege CAPI
C++ / EFL  → docs/cpp-review.md      # RAII, ecore threading
C# NUI     → docs/csharp-nui-review.md # View lifecycle, async
