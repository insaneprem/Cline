What is MCP?

Model Context Protocol (MCP) is an open standard that allows AI assistants like Cline to securely connect to external tools, data sources, and services. Think of it as a universal plug — instead of writing custom integrations for every tool, MCP provides one consistent interface that any compatible AI can use.

In our organisation, MCP servers expose internal capabilities — such as [describe your tools here, e.g. project management, internal APIs, databases] — directly to Cline inside your IDE, making your AI assistant organisation-aware.

Security Precautions

Using MCP tools in an organisational context comes with responsibilities. Please follow these guidelines at all times:

  • 🔐
    Never share your auth tokens. Your MCP access tokens are personal credentials. Do not paste them in Slack, emails, or code commits. Store them securely using your OS keychain or a secrets manager.
  • ✅
    Only use approved MCP servers. Do not add external or third-party MCP servers without IT approval. Unapproved servers may exfiltrate data or inject malicious instructions.
  • 👁️
    Review every tool call before approving. Inspect the parameters Cline proposes. A manipulated prompt could trigger unintended data access or actions.
  • 📵
    Disable unused tools. Only keep tools enabled that you actively need. Reducing the active tool surface lowers the risk of accidental or malicious misuse.
  • 🌐
    Use VPN when connecting to internal MCP servers. All internal servers require you to be on the corporate VPN. Do not attempt to access them over public networks.
  • 🚨
    Report suspicious behaviour immediately. If Cline suggests unusual tool calls, accesses unexpected data, or behaves anomalously — disconnect the server and contact [IT Security Contact / Slack channel].
  • 📝
    All tool calls are logged. MCP interactions are audited. Use tools only for legitimate work purposes in line with our Acceptable Use Policy.
🛑
Prompt Injection Warning Be cautious when using MCP tools on content from untrusted sources (emails, web pages, user-generated text). Malicious content may attempt to hijack Cline's tool calls. Always verify context before approving.
